Getting Started with Darkwood (Monitoring API)

A simple API for your BP2S monitoring needs

This application was made using NodeJS

How this app works

  • You provide us with what you want, and that is all you will get.
  • All of the information comes from Peach but it's stored kept and maintained by Darkwood in a separated infrastructure
  • This API can be used for monitoring purposes but also to retrieve information regarding any artifact
  • All output will always be a JSON object
  • This is an API, so please consider it as such.
  • This is a work in progress, so bear with us.
  • We will add more and more functionalities according with your needs.
  • Understand that some operations take more time than other.

Websphere Operations

    Websphere
  • /api/:server/websphere/, get all available metrics for that server regarding Websphere(this includes IHS server).
  • /api/:server/websphere/:trigram?/, all websphere metrics for the specific trigram
  • /api/:server/websphere/:trigram?/:(was-trigram-n || https-trigram)?/, all WAS or HTTP metrics for the specific trigram
  • /api/:server/websphere/:trigram?/:(was-trigram-n || https-trigram)?/?last_status=:status, gets you the last time the was or http had the selected status. Choices:["STOPPED","RUNNING","STARTED"] limit will always be 1. (Not compatible with /:resource?)
  • /api/:server/websphere/:trigram?/:(was-trigram-n || https-trigram)?/:resource?, specific resource (status,jdbc,jvm...) was or http metric for the specific trigram

Host Operations

    Hosts
  • /api/hosts/websphere/, get all current hosts that have Websphere instances running plus the enviroment it's in. (Always returns a nameless list)

Optional Arguments

    Common operations
  • limit=N retrives N results, the default is 1, the result value will always be a list (even if empty)

Hands on examples

  • /api/s00vl9937872/websphere/bch/was-bch-1/status This would get you the status of BCH WAS server in the s00vl9937872 host
  • /api/s00vl9937872/websphere/bch/was-bch-1/status/?limit=3 This would get you the last 3 status of BCH WAS server in the s00vl9937872 host
  • /api/s00vl9937879/websphere/bch/was-bch-1/?last_status=STOPPED Will return the datetime of the last time the was-bch-1 instance in the s00vl9937879 host was STOPPED (not case sensetive)
